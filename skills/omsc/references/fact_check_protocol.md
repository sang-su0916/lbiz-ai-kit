# OMSC 팩트체크 프로토콜

## 요약 (500자 이내)

새 스킬을 만들 때 **법령 조문·요율·한도·과표 구간·고시 값**은 반드시 정부 원문을 `WebFetch` 하여 교차 확인한 뒤 `SKILL.md` 와 `calculator.py` 에 하드코딩합니다. Claude 기존 지식·언론·위키 단일 출처는 **불인정**. 원문 접근이 SPA 등으로 실패하면 2차 출처 명시 + 상수님(도메인 전문가)에게 확인 요청 → `SKILL.md` "검증 기록" 에 `YYYY-MM-DD 확인자: XXX · 출처: URL` 기록. 모든 인용은 URL + 발췌문 ≤125자 + 조회일자를 함께 남깁니다. 이 프로토콜은 hallucination 방지의 최종 방어선입니다.

## 상세 흐름 (500자 이내)

1. **원문 시도**: `WebFetch https://www.law.go.kr/법령/{법명}` → 해당 조문 본문 확인. 조문 번호·항·호까지 정확히 인용.
2. **교차 확인**: 같은 수치를 2차 소스 1개 이상에서 재확인 (공식 해설서·실무 가이드). 불일치 시 개정 이력 확인.
3. **SPA 실패 대응**: `law.go.kr` 등이 JS 렌더링으로 fetch 실패 → `WebSearch "법명 §조항 site:law.go.kr"` 로 정적 미러 URL 탐색. 그래도 실패 시 4단계로.
4. **사용자 확인**: 상수님께 "다음 수치를 확인 부탁드립니다: [수치·근거 후보 URL]" 메시지로 검증 요청. 답변 받으면 `SKILL.md` "검증 기록" 표에 기록.
5. **인용 형식 준수**: `> 조문 발췌 ≤125자 — 출처: URL (YYYY-MM-DD 조회)` 형식으로 `SKILL.md` "법령 근거" 섹션에 남김. 발췌는 원문 그대로 (번역·의역 금지).

```markdown

| ----------- | ------------------------------- | --------------------------------------------------------------------------- |
| 근기법 §56① | 연장근로 통상임금 50% 이상 가산 | [law.go.kr](https://www.law.go.kr/법령/근로기준법/제56조) (2026-04-13 조회) |
```

### 본문 인용 (블록 인용문)

```markdown
> "사용자는 연장근로에 대하여 통상임금의 100분의 50 이상을 가산하여 지급하여야 한다."
> — 근로기준법 §56①, 출처: https://www.law.go.kr/법령/근로기준법 (2026-04-13 조회)
```

**규칙**

- 발췌문 ≤125자 (원문 그대로 복사)
- URL 전체 경로 (shortener 금지)
- 조회일자 `YYYY-MM-DD` 형식
- 조문 번호·항·호 정확히 (§56①, §56② 전단 등 구분)

## 실패 시 대체 흐름

```
WebFetch 원문
  └ 실패 (SPA · 404 · 접근 차단)
```
